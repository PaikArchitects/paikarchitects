# MOBILE_FIX_SPEC

**목적:** 모바일(<768px) 전용 3개 항목을 수정한다. M1(필터 진입 후 표시), M2(필터 칩 화살표 어포던스 + 마우스 드래그), M3(최초 프로젝트 상단 정렬). 데스크톱·태블릿 코드는 일절 건드리지 않는다.

**대상 파일**
- `src/components/MobileProjectWall.tsx`
- `src/components/LandingExperience.tsx` (M1의 모바일 필터 노출 조건만)

**동결 보호 조항:** 모바일 인터랙션 레이어(셔플, 확장 모프, FLIP, 트랙 스냅)는 v4.1로 동결 대상이다. 이 스펙은 셔플·확장 모프의 스크롤 타이밍 로직을 **수정하지 않는다.** M3는 "최초 진입 시 초기 스크롤 위치"만 바꾸며, active 확장 시의 수직 중앙 정렬(`availCenter` 기반)은 그대로 둔다.

**검증은 `npx tsc --noEmit`만 사용.** `npm run dev` / `npm run build` 금지.

## M1 — 필터를 진입(WORK) 후 표시

**현상:** 모바일에서 필터 칩 행이 최초부터 떠 있다. 데스크톱처럼 WORK 진입 시 표시되어야 한다.

**현재 구조:**
- `LandingExperience`의 `showFilters` 상태가 `/work` 진입 시 `initialShowFilters=true`로 켜지고, 모바일에도 그대로 전달된다.
- `MobileProjectWall`의 칩 행(`position: sticky; top: 0; height: CHIPS_H`)이 조건 없이 렌더된다.

**변경:**
1. `MobileProjectWall`이 필터 칩 행을 렌더할지 여부를 `showFilters`(또는 동등한 prop)로 제어하도록 한다. `LandingExperience`에서 `MobileProjectWall`로 `showFilters` 값을 prop으로 전달한다 (이미 전달 중이면 재사용).
2. 칩 행 컨테이너 (`<div style={{ position: 'sticky', top: 0, zIndex: 5, height: CHIPS_H, ... }}>`) 를 `showFilters`가 true일 때만 렌더한다.
3. **CHIPS_H 예약 연동 (중요):** 칩 행이 숨겨지면 그 높이(44px)가 사라진다. 수직 정렬·셔플 스크롤 계산에서 `CHIPS_H`를 상수로 더하는 모든 지점(`availCenter = CHIPS_H + ...`, `availCenter` 류 2곳)이 칩 행 표시 여부와 일치해야 한다.
   - 칩 행 숨김 시: `availCenter` 계산의 `CHIPS_H` 항을 0으로 (또는 실제 칩 행 높이를 동적으로 측정).
   - **단, M3 적용 후에는 최초 진입이 상단 정렬이므로 이 중앙 정렬 식은 active 확장 시에만 쓰인다.** active 확장은 항상 WORK 진입(=필터 표시) 이후이므로, 보수적으로는 active 시점엔 `showFilters`가 켜져 CHIPS_H가 유효하다고 가정 가능. 구현 시 이 가정이 성립하는지(랜딩 `/`에서 바로 카드 탭→확장 시 필터가 꺼져 있는 경로) 반드시 확인하고, 안 맞으면 `availCenter`를 칩 행의 실제 렌더 높이로 동적 산출할 것.

**경계 케이스:** 루트(`/`)에서 카드를 바로 탭해 확장에 진입하는 경로. 이때 필터는 꺼져 있어야 하나(M1), 확장 중앙 정렬은 CHIPS_H를 가정한다. → `availCenter`의 CHIPS_H 항을 `chipsVisible ? CHIPS_H : 0`으로 분기.

## M2 — 필터 칩 오버플로 화살표 + 마우스 드래그

**현상:** 필터 칩이 한 화면에 다 안 보이는데, 더 있다는 표시(화살표)가 없다. 또한 데스크톱 브라우저를 모바일 폭으로 줄였을 때 마우스 드래그로 칩을 스크롤할 수 없다.

**현재 구조:**
- `chipFade` 상태(`{ left, right }`)가 이미 존재하여 좌우 오버플로를 감지한다 (`scrollLeft` 기반).
- 그러나 (a) 명시적 화살표 기호가 없고, (b) 칩 행이 네이티브 스크롤(`overflow-x`)이라 마우스 드래그 미지원.

**변경:**

### M2-1. 화살표 어포던스
- 기존 `chipFade.right`가 true일 때, 칩 행 우측에 **그라데이션(32px, #FFFFFF→투명) + 화살표 글리프(`›` 또는 chevron)** 를 오버레이한다.
- `chipFade.left`가 true일 때 좌측에도 대칭 적용 (선택).
- 화살표는 칩 위에 `position: absolute`, `pointer-events: none` (스크롤 방해 금지), 수직 중앙.
- 끝까지 스크롤하면 해당 방향 화살표·그라데이션 사라짐 (기존 chipFade 로직 그대로 활용).
- **이 화살표 구현은 RESPONSIVE_UNIFY_SPEC PART D의 데스크톱 필터 바 화살표와 시각적으로 동일한 기호·스타일을 사용한다** (일관성).

### M2-2. 마우스 드래그-투-스크롤 (혼용 환경)
- 칩 행에 Pointer 기반 드래그-투-스크롤을 추가한다. 패턴은 `ContentArea`의 드래그 핸들러(`onPointerDown`/`onPointerMove`/`onPointerUp`, 5px 이동 임계, `setPointerCapture`)를 참조.
- 칩 행은 네이티브 터치 스크롤(`overflow-x: auto` + `touch-action: pan-x`)을 유지하되, **마우스 입력에 한해** Pointer 드래그로 `scrollLeft`를 조작한다:
  - `onPointerDown`: `e.pointerType === 'mouse'`일 때만 드래그 시작, `startX`·`startScrollLeft` 기록, `setPointerCapture`.
  - `onPointerMove`: 드래그 중이면 `el.scrollLeft = startScrollLeft - (e.clientX - startX)`.
  - `onPointerUp`: 드래그 종료. 이동량 < 5px이면 칩 클릭(필터 선택)으로 처리되도록 `onClick`을 막지 않는다 (5px 이상 이동 시에만 클릭 억제).
- **터치는 네이티브 스크롤에 맡기고 Pointer 드래그를 적용하지 않는다** (`pointerType !== 'mouse'` 시 early return). 이중 스크롤 충돌 방지.

## M3 — 최초 프로젝트 상단 정렬

**현상:** 모바일 최초 진입 시, 그리고 필터에서 프로그램 선택 시, 첫 프로젝트가 상단에 여백을 두고 (수직 중앙쯤에서) 시작한다. 데스크톱처럼 썸네일이 **상단 정렬**되어야 한다.

**원인:** 피드(`containerRef`)의 초기 `scrollTop`이 0이 아니거나, 콘덴스드 월의 첫 아이템 위에 중앙 정렬용 여백이 예약돼 있다. 확장 시 `availCenter` 기반 중앙 정렬과 **비확장(월) 상태의 정렬은 별개**다 — 월 상태에서 첫 d=0 아이템이 뷰포트 상단에 붙어야 한다.

**변경:**
1. **비확장(월) 상태의 첫 아이템 상단 정렬:** 피드 컨테이너 내부, 콘덴스드 월의 첫 아이템 이전에 삽입된 상단 스페이서/여백(있다면)을 제거하거나, 초기 `scrollTop = 0`에서 첫 d=0 아이템이 뷰포트 최상단(필터 칩 행 바로 아래, 칩 숨김 시 헤더 바로 아래)에 오도록 한다.
2. **필터 선택 후 동일:** 필터 변경으로 목록이 교체될 때도 첫 아이템이 상단 정렬되도록, 필터 적용 후 `scrollTop = 0` 리셋 + 상단 여백 없음을 보장한다.
3. **active 확장 시의 중앙 정렬은 유지:** `availCenter` 기반 확장 스크롤(L740 부근, L881 부근)은 **변경하지 않는다.** 이건 선택된 프로젝트를 가용 영역 중앙에 두는 의도된 동작이고 동결 대상이다. M3는 어디까지나 "월(비확장) 상태의 첫 진입·필터 후 정렬"만 상단으로 바꾼다.

**구현 주의 (동결 충돌 회피):**
- 콘덴스드 월의 수직 중앙 기준 스크롤 반응(터치 hover 대체)과 셔플(6초 idle → 랜덤 프로젝트 중앙 스크롤)은 v4.1 동결 로직이다. M3의 "상단 정렬"은 **초기 정지 위치**에만 적용하고, 셔플이 시작되면(또는 사용자가 스크롤하면) 기존 중앙 기준 위계 로직이 그대로 작동하게 둔다.
- 즉 변경 범위: ① mount 직후 초기 `scrollTop` ② 필터 교체 직후 `scrollTop`. 이 둘만 상단 정렬. 그 외 스크롤 중 위계·셔플은 불변.
- 만약 첫 아이템 상단 정렬이 콘덴스드 월의 "뷰포트 수직 중앙 기준 d=0 판정"과 시각적으로 어색하게 충돌하면(첫 아이템이 상단인데 d=0 강조는 중앙 기준), 이 충돌 지점을 구현 후 화면 녹화로 확인할 항목으로 보고할 것. 임의로 위계 판정 로직을 바꾸지 말 것.
