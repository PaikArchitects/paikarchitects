# MOBILE_SPEC v4 — 모바일 월 재설계 + 트랙 보완 (v3 배포 피드백 8건 반영)

> v3 구현의 골격(월 우선 진입 / 인라인 트랙 / 헤더 컴팩트화)은 **유지·승인되었다.**
> 이 스펙은 그 위에서 8개 보완을 수행한다. 핵심은 §A — 모바일 월을
> "균일 카드 피드"에서 **데스크톱 ProjectWall과 동일한 문법의 콘덴스드 월**
> (크기 위계 + 스크롤 반응 + 셔플 활동성 + 중앙 정렬)로 재설계하는 것이다.
> 데스크톱(≥768px) 동작은 일절 변경하지 않는다.
>
> 구현 완료 후 `npx tsc --noEmit` 검증. `npm run dev` / `npm run build` 금지.

---

## A. 모바일 월 재설계 (피드백 1·2·3)

현재의 풀폭 3:2 카드 리스트를 폐기하고, 데스크톱 월의 축소·중앙정렬 버전으로 교체한다.

### A-1. 아이템 구조 — 페어 문법 승계 + 중앙 정렬
- 데스크톱과 동일한 **[텍스트(좌) | 썸네일(우)] 페어** 구조. 텍스트는 우정렬로 썸네일에 flush (간격 8px) — 데스크톱 문법 그대로.
- 데스크톱은 우측 콘텐츠 영역 때문에 월이 좌측 배치였으나, 모바일은 월 단독 화면이므로 **페어 전체를 수평 중앙 정렬**한다. 중앙 기준 = 썸네일 좌측 에지가 아니라 **페어 묶음의 중심**.
- 텍스트: 프로젝트명 13px / w400 / keep-all 2줄 허용, 그 아래 카테고리 9px 대문자 / opacity 0.45. 텍스트 블록 최대 폭 130px.

### A-2. 3단 크기 위계 + 스크롤 반응 (핵심)
- 썸네일 3:2 (`cldCard(coverImage, 480)`), 3단 위계:

| 티어 | 조건 | 썸네일 크기 | opacity |
|---|---|---|---|
| d=0 | 위계 중심 | **w192 × h128** | 1 |
| d=1 | 중심 ±1 | w150 × h100 | 0.45 |
| d≥2 | 그 외 | w114 × h76 | 0.3 |

- width·height·opacity transition **400ms ease** (데스크톱과 동일).
- **위계 중심 = 뷰포트 수직 중앙에 가장 가까운 아이템.** 스크롤 이벤트(rAF 스로틀)로 매 프레임 판정 — 사용자가 위아래로 스크롤하면 중앙을 지나는 썸네일이 차례로 커졌다 작아진다. 터치 환경의 호버 등가물이 이 스크롤 반응이다.
- 위 크기표는 **`WALL_SCALE` 단일 상수(기본 1.0)로 전체 배율 조정** 가능하게 구현할 것 — 1차 배포 후 시각 검증으로 배율을 확정한다.
- 아이템 간 수직 간격 **14px**. 한 화면에 5~6개 페어가 동시에 보이는 밀도가 목표 — 이 밀도가 상하 연결성(월의 컬럼감)을 만든다. 피드 상하단에 `50vh` 패딩을 두어 첫/마지막 아이템도 뷰포트 중앙에 올 수 있게 한다.

### A-3. 셔플 부활 — 월 결합형 (v3의 셔플 폐지를 대체)
- v3에서 폐지했던 셔플을 **월 하이라이트 순환**으로 재정의해 부활한다 (데스크톱 Idle의 "월 자동 스크롤 + 동기 하이라이트" 문법의 이식).
- 동작: 사용자 무조작 상태에서 **6초마다** 랜덤 프로젝트(셔플 큐 — 데스크톱과 동일한 큐 로직 재사용 가능)로 하이라이트 이동 → 해당 아이템을 뷰포트 중앙으로 **smooth 스크롤** → 위계 중심이 따라 이동하며 d=0으로 확대.
- 일시정지 조건: 터치 시작·스크롤·active 진입 시 **즉시 정지**. 무조작 **8초** 경과 후 재개. 필터 변경 시 큐 재생성(기존 로직). 프로그래매틱 스크롤이 일시정지 판정을 다시 트리거하지 않도록 가드할 것.
- 블랙아웃/페이드 없음 — 모바일 월에는 히어로 이미지가 없으므로 스크롤 이동 자체가 활동성이다.

## B. Active 모프 재정의 (피드백 2·4)

### B-1. 성장 모프
- 탭 시 해당 썸네일이 **풀폭 히어로**(폭 = 100vw − 32px, 3:2, `cldCard(…, 800)`)로 **부드럽게 성장** — 500ms `cubic-bezier(0.7,0,0.3,1)`. 작은 썸네일 → 큰 히어로의 스케일 점프가 이 인터랙션의 드라마이므로, 크기·위치를 모두 보간하는 연속 모프여야 한다 (FLIP 기법 권장: 썸네일의 시작 rect → 히어로의 종료 rect).
- 페어의 텍스트 블록은 성장 시작과 동시에 페이드 아웃 (200ms).

### B-2. 수직 중앙 정렬 열람 위치 (피드백 4)
- 탭 위치가 화면 어디든, 확장 완료 시 **확장 블록(BACK 행 + 타이틀 + 트랙 + 카운터)의 수직 중심이 가용 영역(헤더 56px + 칩 행 아래 ~ 뷰포트 하단)의 수직 중심에 정렬**되도록 프로그래매틱 스크롤을 모프와 **동시 진행**한다 (동일 500ms, 동일 easing).
- 수축(BACK) 시 역모프 + 해당 아이템이 위계 중심에 오는 위치로 복귀 스크롤.

### B-3. 정보 슬라이드 위치 — BIG 문법 (피드백 2 후단)
- 트랙 슬라이드 순서를 변경한다: **[① 히어로 이미지] → [② 정보 슬라이드] → [③ 이후 슬라이드들…]**
- 데스크톱에서 좌측(트랙 선두)에 있던 정보 슬라이드가, 모바일에서는 **메인 이미지 오른쪽 첫 번째**로 온다 (BIG 모바일 동일). 정보 슬라이드 내용: 위치 / TYPOLOGY / STATUS / YEAR (현 데이터 유지), 폭 60vw, 수직 중앙.
- 초기 스크롤 위치 = 히어로가 화면에 정렬된 0 지점.

## C. 트랙 수직 고정 + 스냅 + 다이어그램 (피드백 5·6)

### C-1. 수직 요동 제거 (피드백 5)
- **트랙의 모든 수직 치수를 상수화한다.** 히어로 높이 `H = (100vw − 32px) × 2/3` 기준:
  - 이미지·정보·크레딧 슬라이드 높이 = **H** (상하 라인 일치)
  - 다이어그램(isDiagram) 슬라이드 높이 = **H × 2/3**, 트랙 내 수직 중앙 (데스크톱 72%/48% 비례와 동일)
  - 캡션 영역 = 트랙 하단에 **항상 28px 고정 예약** (캡션 유무·길이와 무관). 캡션은 이 영역 안에서만 렌더(10px, 최대 2줄, 넘치면 ellipsis).
- 트랙 컨테이너: `touch-action: pan-x`, `overflow-y: hidden`, `overscroll-behavior-x: contain`. **트랙 위 제스처가 피드의 수직 스크롤로 전파되지 않아야 하며, 어떤 슬라이드 조합에서도 트랙 블록의 총 높이가 변하지 않아야 한다.**

### C-2. 스냅 복원 (피드백 6 후단)
- `scroll-snap-type: x mandatory`, 각 슬라이드 `scroll-snap-align: center`, 컨테이너 `scroll-padding-inline: 16px`. 스와이프가 항상 슬라이드 중앙 정렬로 안착한다.
- 카운터(`NN / NN`)는 스냅 안착 슬라이드 기준으로 갱신.

### C-3. 다이어그램 탭 진행 복원 (피드백 6 전단)
- diagramSet 슬라이드 **탭 = 다음 서브슬라이드** (루프) + 자동 진행 타이머 리셋. 자동 진행 3000ms 유지. 탭 판정은 수평 이동 5px 미만의 터치만 (스와이프와 구분).

## D. 필터 칩 보완 (피드백 7·8)

### D-1. 오버플로 어포던스 (피드백 7)
- 칩 행 우단에 **32px 화이트→투명 그라디언트 오버레이** — 우측으로 더 스크롤 가능할 때만 표시(스크롤 위치 감지로 토글, 끝 도달 시 페이드 아웃). 좌측도 대칭 적용(좌로 스크롤된 상태일 때).
- 우측 패딩을 조정해 마지막 가시 칩이 **의도적으로 절반쯤 잘려 보이게** (peek) — 잘림 자체가 스크롤 가능성의 1차 신호, 그라디언트가 2차 신호.

### D-2. 폴드 전환 (피드백 8)
- 필터 선택 시 2단계 시퀀스:
  1. **접힘**: 현재 모든 아이템이 위로 접힘 — height → 0, opacity → 0, 280ms, 위→아래 stagger 15ms. (active 열려 있으면 동시에 해제)
  2. **펼침**: 필터 결과 아이템들이 위→아래로 펼쳐짐 — height 0 → 티어값, opacity 0 → 1, 400ms ease-out, stagger 50ms (상한 600ms).
- 접힘 완료 후 80ms 정지 → 펼침 시작 (두 단계가 겹치지 않도록 시퀀싱).
- 인트로 직후 최초 등장도 2단계(펼침)와 동일 모션으로 통일 — v3의 translateY 캐스케이드를 이 펼침 모션으로 교체.

## E. 불변 사항

- 데스크톱(≥768px) 전체 / 헤더 컴팩트 구조(56px 불투명 바, 워드마크 22px, 내비 우측 정렬) / URL·popstate·딥링크 규칙(v3 §2-D) / `cldCard` 헬퍼 / BACK 행·타이틀 행·카운터 행 구성은 v3 그대로 유지.
- 딥링크 직접 진입 시에도 §B-2의 수직 중앙 정렬 위치로 로드 (모프 생략, 즉시 상태).

## F. 검증 체크리스트 (실기기 세로 녹화)

1. 월: 한 화면 5~6 페어, 중앙 정렬, 스크롤에 따라 중앙 통과 썸네일이 확대/축소 (400ms 부드러움)
2. 셔플: 무조작 6초마다 랜덤 프로젝트로 자동 스크롤 + 중앙 확대, 터치 즉시 정지, 8초 후 재개
3. 탭: 썸네일 → 풀폭 히어로 연속 성장 모프 (점프·깜빡임 없음)
4. 정렬: 탭 위치 무관하게 확장 블록이 화면 수직 중앙에 안착
5. 정보 슬라이드: 히어로 **오른쪽 첫 번째**에 위치
6. 트랙: 좌우 스와이프 시 수직 요동 0px (캡션 유무 무관), 트랙 위 제스처가 피드 수직 스크롤로 새지 않음
7. 스냅: 스와이프 안착이 항상 슬라이드 중앙 정렬, 카운터 동기 갱신
8. 다이어그램: 탭으로 서브슬라이드 진행 + 3초 자동 진행 공존
9. 칩: 우측 그라디언트 + peek로 추가 항목 인지 가능, 끝 도달 시 그라디언트 소멸
10. 필터: 전체 접힘 → 해당 필터만 펼침 시퀀스, active 동시 해제
11. BACK / 브라우저 뒤로가기 / 딥링크: v3 규칙대로 동작
12. 데스크톱: 전부 불변
